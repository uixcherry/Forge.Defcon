# Forge Defcon

PC scanner for detecting cheats, prohibited software, and DMA hardware. Built with WPF (.NET 9).

## Руководство для администраторов

Документация для администраторов античита: что делает каждый детектор и как правильно выявлять читеров.

### Общие рекомендации по выявлению читеров

1. **Запускайте полное сканирование** — все 20 модулей дают полную картину.
2. **Обращайте внимание на Critical** — это прямые улики (процессы, драйверы, DMA, файлы).
3. **High** — косвенные признаки (история браузера, автозагрузка, исключения Defender).
4. **Cleanup Detector** — если сработал, читер мог затирать следы перед проверкой.
5. **Комбинация угроз** — несколько детекторов по одной теме (например DMA + File + Device) почти всегда означают использование читов.

## Справочник детекторов

### Process Scanner
**Что ищет:** Запущенные процессы читов, отладчиков, инжекторов, тренеров, HWID-спуферов.

**Как работает:** Перечисляет все процессы Windows, сверяет имена и заголовки окон с базой 140+ известных инструментов (Cheat Engine, x64dbg, IDA, Extreme Injector, PCILeech, WeMod, HWID Spoofer и т.д.). Проверяет процессы, запущенные из Temp.

**Как интерпретировать:** Любое совпадение — **Critical**. Если процесс найден, читер либо использует его сейчас, либо забыл закрыть перед проверкой.

---

### Driver Scanner
**Что ищет:** Уязвимые/эксплойтные kernel-драйверы (LOLDrivers), используемые для обхода античита.

**Как работает:** Сканирует `System32\drivers`, сверяет имена файлов с базой 80+ драйверов (kdmapper, dbk64, capcom.sys, mhyprot, pcileech.sys и т.д.).

**Как интерпретировать:** Найденный драйвер — **Critical**. Такие драйверы часто используются для загрузки читов в ядро или обхода защиты.

---

### DMA Scanner
**Что ищет:** DMA-оборудование и софт (PCILeech, Screamer, Squirrel, FPGA-платы).

**Как работает:** Сканирует PCIe через WMI (Xilinx, Altera, FT601, fake VID), Thunderbolt/USB4, USB DMA-адаптеры, ищет файлы (pcileech, screamer, .bit, .rbf), ключи реестра, проверяет VT-x/IOMMU.

**Как интерпретировать:** Любая находка — **Critical**. DMA-читы читают память игры с отдельного ПК/платы, античит их почти не видит. Наличие FPGA или FT601 — сильный признак.

---

### File Scanner
**Что ищет:** Исполняемые файлы читов, .ct/.cetrainer, подозрительные расширения, папки с cheat-названиями.

**Как работает:** Сканирует Desktop, Downloads, Program Files, Temp и др. Сверяет имена файлов и расширения с чёрным списком (cheatengine.exe, pcileech.exe, aimbot, dma_cheat и т.д.).

**Как интерпретировать:** **Critical** — известные .exe читов. **High** — подозрительные паттерны в именах. Учитывайте false positives в путях разработки (Visual Studio, Python и т.д.) — они в белом списке.

---

### Registry Scanner
**Что ищет:** Установленные читы (Uninstall), UserAssist/BAM (история запусков), отладчики, изменения политик безопасности.

**Как работает:** Сканирует Uninstall, Software, UserAssist, BAM, MountPoints, ключи отладчиков.

**Как интерпретировать:** Uninstall-записи — **Critical** (софт был установлен). UserAssist/BAM — **High** (запускался). Отладчики в реестре — **Critical**.

---

### Memory Scanner
**Что ищет:** DLL читов, загруженные в процессы (speedhack, aimbot, d3dhook и т.д.).

**Как работает:** Перечисляет модули всех процессов (`Process.Modules`), сверяет имена и пути с базой cheat-DLL.

**Как интерпретировать:** Найденная DLL — **Critical**. Чит активен в памяти. Исключены системные и легитимные DLL.

---

### Device Scanner
**Что ищет:** DMA USB-устройства (Xilinx, Altera, FT601/FT600), FPGA-адаптеры, подозрительные PnP-устройства.

**Как работает:** WMI `Win32_PnPEntity` — проверка Vendor ID, Device ID, имён устройств. FT601 (0403:6010) = Screamer, Xilinx = FPGA.

**Как интерпретировать:** **Critical** — FT601, Xilinx Platform Cable и т.п. **High** — другие FPGA/FTDI. Учитывайте легитимные устройства (Arduino и т.д. исключены где возможно).

---

### VM / Sandbox Scanner
**Что ищет:** Виртуальные машины (VMware, VirtualBox, Hyper-V, QEMU, Sandboxie).

**Как работает:** WMI (BIOS, оборудование), процессы, драйверы, MAC-адреса.

**Как интерпретировать:** **Info–Medium**. VM может означать обход (игра в VM, античит на хосте) или легитимное использование. Требует контекста.

---

### Steam Account Scanner
**Что ищет:** Steam-аккаунты, когда-либо входившие на ПК.

**Как работает:** Парсит loginusers.vdf, userdata, реестр. Выдаёт SteamID, persona, имя.

**Как интерпретировать:** **Info** — один аккаунт. **Medium** — 3+ аккаунтов (возможный шаринг или мультиакки).

---

### Cleanup Detector
**Что ищет:** Признаки затирания следов перед проверкой (Prefetch, Event Log, CCleaner и т.д.).

**Как работает:** Проверяет количество Prefetch vs uptime, события очистки логов (104, 1102), недавний запуск CCleaner/BleachBit/Privazer, пустые Recent Files, отключённый System Restore.

**Как интерпретировать:** **Critical** — Prefetch очищен, логи стёрты, cleanup-утилита запущена недавно. Сильный признак того, что пользователь пытался скрыть активность.

---

### Isolation Scanner
**Что ищет:** Отключённый Firewall, Defender, Secure Boot, UAC, Test Signing.

**Как работает:** Реестр (политики Firewall/Defender), WMI (Secure Boot), проверка UAC и подписи драйверов.

**Как интерпретировать:** **Medium**. Ослабленная защита облегчает работу читов. Не доказывает читерство, но повышает подозрения.

---

### Startup Scanner
**Что ищет:** Автозагрузка читов (Run/RunOnce, Startup, Scheduled Tasks).

**Как работает:** Сканирует Run/RunOnce, папки автозагрузки, планировщик задач. Сверяет с cheat-паттернами.

**Как интерпретировать:** **High**. Чит настроен на автозапуск. Проверьте путь и имя исполняемого файла.

---

### Network Scanner
**Что ищет:** Блокировка античита в hosts, подмена proxy/DNS, подозрительные VPN.

**Как работает:** Читает hosts, проверяет proxy/DNS в реестре, перечисляет сетевые адаптеры.

**Как интерпретировать:** **Critical** — домены античита в hosts. **Medium** — proxy/DNS. Может означать попытку обхода проверки или блокировки.

---

### Defender Exclusions
**Что ищет:** Исключения Defender для путей/процессов/расширений, связанных с читами.

**Как работает:** Сканирует реестр исключений Defender, ищет cheat-ключевые слова.

**Как интерпретировать:** **Critical** — путь к читу в исключениях. **Info** — общие исключения. Прямая улика, что пользователь намеренно отключил защиту для читов.

---

### USB History
**Что ищет:** История подключения атакующих USB (Rubber Ducky, Bash Bunny, Teensy, Flipper).

**Как работает:** USBSTOR, MountedDevices, недавние USB в реестре.

**Как интерпретировать:** **Critical**. Может означать использование HID-атак или скриптов. Требует контекста (разработка, пентест).

---

### Execution History
**Что ищет:** История запуска читов (Prefetch, AmCache, MuiCache, RecentApps).

**Как работает:** Сканирует Prefetch, AmCache, MuiCache, RecentApps на имена cheat-инструментов.

**Как интерпретировать:** **Critical**. Даже если процесс не запущен, софт запускался ранее. Сильная улика.

---

### Browser History
**Что ищет:** Посещение cheat-сайтов (UnknownCheats, MPGH, cheat engine и т.д.) в Chrome/Edge/Brave/Firefox.

**Как работает:** SQLite History (Chromium), places.sqlite (Firefox). Поиск по доменам и ключевым словам.

**Как интерпретировать:** **High**. Косвенный признак. Сам по себе не доказывает использование читов, но в сочетании с другими находками — весомый аргумент.

---

### HWID Spoofer
**Что ищет:** Процессы, файлы, реестр, драйверы HWID-спуферов (AntiFakHWID, volumeid и т.д.).

**Как работает:** Процессы, файлы, ключи реестра, имена драйверов.

**Как интерпретировать:** **Critical**. Спуферы используются для обхода HWID-банов. Прямая улика.

---

### Kernel Debug
**Что ищет:** Test signing, отключённая проверка подписи драйверов, kernel debugger.

**Как работает:** Реестр CI Policy, BCD.

**Как интерпретировать:** **High**. Облегчает загрузку неподписанных драйверов (читов). Не доказывает читерство, но типично для читеров.

---

### Game Cheats
**Что ищет:** Файлы/папки читов внутри каталогов игр (Valorant, CS2, Fortnite и т.д.).

**Как работает:** Сканирует известные пути установки игр на наличие .ct, loader, aimbot и т.п.

**Как интерпретировать:** **High**. Чит лежит рядом с игрой — очень сильный признак.

---

### Suspicious Services
**Что ищет:** Windows-сервисы с cheat-именами (kdmapper, spoofer, inject и т.д.).

**Как работает:** Перечисляет все сервисы, сверяет имена и пути к бинарникам.

**Как интерпретировать:** **High**. Чит или вспомогательный софт зарегистрирован как сервис.

## Сборка и запуск

```bash
dotnet restore
dotnet build
dotnet run
```

Требуется **Windows** и **.NET 9 SDK**.

### Публикация (Release)

```bash
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o ./publish
```

Готовый `Forge.Defcon.exe` будет в папке `publish`.
